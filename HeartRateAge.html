Introduction

Every person in the world should exercise, at least, to a certain degree. We were tasked with the objective of writing a program that could identify the percent of max heart rate the person is currently using. This was achived through the implementation of linear functions that defined the boundaries of certain percentage regions. Each function has been extrapolated from the available graphics. Although we defined both top and lower boundaries, we will only employ the higher boundaries for our analysis. (These should be given emphasis in the final analysis.) In order to account for out of sample errors, we assigned an extra category which accounts for BPM theoretically above 100% or extremely near the limit of the highest linear function defined.

Data Gathering

We will first define the parameters needed to classify BPM and Age in accordance to the % of Max Heart Rate.

Red Line

The HighRedLine is a pointer for a function of Age that defines the top limit for the red line zone. m = -1

HighRedLine = @(Age) ((205-135)/(15-85))*Age + 205;

% The LowRedLine is a pointer for a function of Age that defines the lower
% limit for the red line zone. m = -6/7

LowRedLine = @(Age) ((185-125)/(15-85))*Age + 125;

Anaerobic

The HighAnaerobic is a pointer for a function of Age that defines the high limit for the Anaerobic Zone. m = -6/7

HighAnaerobic = @(Age) ((185-125)/(15-85))*Age + 125;

% The LowAnaerobic is a pointer for a function of Age that defines the
% low limit for the Anaerobic Zone. m = -11/14

LowAnaerobic = @(Age) ((165-110)/(15-85))*Age + 165;

Aerobic

The HighAerobic is a pointer for a function of Age that defines the high limit for the Aerobic Zone. m = -11/14

HighAerobic = @(Age) ((165-110)/(15-85))*Age + 165;

% The LowAerobic is a pointer for a function of Age that defines the
% low limit for the Aerobic Zone. m = -5/7

LowAerobic = @(Age) ((145-95)/(15-85))*Age + 145;

Energy Efficient/Recovery

The HighEnergy is a pointer for a function of Age that defines the high limit for the Energy Efficient/Recovery Zone. m = -5/7

HighEnergy = @(Age) ((145-95)/(15-85))*Age + 145;

% The LowEnergy is a pointer for a function of Age that defines the
% low limit for the Energy Efficient/Recovery Zone. m = -9/14

LowEnergy = @(Age) ((125-80)/(15-85))*Age + 125;

Healthy Heart

The HighHealthy is a pointer for a function of Age that defines the high limit for the Healthy Heart Zone. m = -9/14

HighHealthy = @(Age) ((125-80)/(15-85))*Age + 125;

% The LowHealthy is a pointer for a function of Age that defines the
% low limit for the Healthy Heart Zone. m = -1/2

LowHealthy = @(Age) ((105-70)/(15-85))*Age + 125;

User Input

The instructions request that the user inputs specific parameters to calculate results. These parameters are BPM and Age.

The input function prints the statement and takes the value of the User-Inputed Integer. For this specific program, or version of the program, I removed the input function call and assigned the validation values directly through a 1 by 4 array for Age and BPM.

Age = [70,50,21,21];

BPM = [90,143,80,220];

Decisions & Parameter Evaluation

We will use selection structures in order to compare the equations developed in a previous section. The selection process will be based on a comparison of the independent value 'Age' vs the provided BPM value.

for ii=1:4
    fprintf('For %2.0f years and %2.0f BPM, \n',Age(ii), BPM(ii))
    if HighHealthy(Age(ii)) > BPM(ii)
        disp('Your heart is running at 50-60% of max heart rate.')
    elseif HighEnergy(Age(ii)) > BPM(ii)
        disp('Your heart is running at 60-70% of max heart rate.')
    elseif HighAerobic(Age(ii)) > BPM(ii)
        disp('Your heart is running at 70-80% of max heart rate.')
    elseif HighAnaerobic(Age(ii)) > BPM(ii)
        disp('Your heart is running at 80-90% of max heart rate.')
    elseif HighRedLine(Age(ii)) > BPM(ii)
        disp('Your heart is running at 90-100% of max heart rate.')
    else
        disp('Your heart is running at 90-100% of max heart rate, or higher!')
    end
end

Conclusion

This program validates the function with an error rate below 95%. The 5% accounts for errors in slope calculations due to innacurate graphical analysis.
